# 浏览器版 SSH 本地工具设计

## 目标

构建一个个人自用的浏览器版 SSH 工具。程序在本机启动 Node.js Web 服务，浏览器访问本地页面后，可以选择已保存连接，一键登录远程服务器或开放 SSH 端口的电脑，并提供终端与远程文件管理能力。

第一版重点是本地使用、连接持久化、凭据加密保存、中文界面和 Rocom Grimoire 风格 UI。

## 范围

第一版包含：

- Node.js 本地网页服务。
- 启动时选择监听 `127.0.0.1` 或 `0.0.0.0`。
- 中文 Web UI。
- 整体视觉遵循 Rocom Grimoire 设计系统。
- 保存连接信息。
- 加密保存密码、私钥和私钥 passphrase。
- 下次选择连接后直接登录。
- SSH 终端。
- SFTP 远程文件浏览。
- 上传文件到远程服务器。
- 下载远程服务器文件。
- 数据保存到程序所在目录下的 `data/`。

第一版不包含：

- 多用户系统。
- 云端同步。
- 公网部署。
- 命令审计。
- 会话录像。
- 左右双栏本地/远程文件管理。
- 批量上传下载。
- 复杂权限系统。

## 技术方案

前端：

- Vite + React。
- xterm.js 渲染终端。
- xterm-addon-fit 处理终端尺寸适配。
- Tab 页面结构：连接管理、终端、远程文件、设置。

后端：

- Node.js。
- Express 提供 HTTP API。
- ws 提供终端 WebSocket。
- ssh2 提供 SSH shell 和 SFTP 能力。
- busboy 或 multer 处理上传流。

数据存储：

```text
程序目录/
  data/
    connections.json
    vault.enc
    app.key
```

`connections.json` 保存非敏感连接信息，例如连接名称、host、port、username、认证方式和 credentialId。

`vault.enc` 保存加密后的密码、私钥内容和 passphrase。

`app.key` 保存本地加密密钥。该方案满足下次直接登录的体验。安全取舍是：如果他人复制完整程序目录，理论上可以解密保存的凭据。

## 启动流程

程序启动时在命令行提示：

```text
请选择监听地址：
1. 127.0.0.1 仅本机访问（推荐）
2. 0.0.0.0 局域网访问（谨慎）
```

选择 `127.0.0.1` 时，只允许本机访问。

选择 `0.0.0.0` 时，允许局域网设备访问。程序必须显示警告：

```text
警告：0.0.0.0 会允许局域网设备访问本工具。
请只在可信网络中使用，并妥善保管启动链接中的 token。
```

启动后生成随机 token，并打开浏览器访问：

```text
http://127.0.0.1:8765/?token=随机token
```

如果监听 `0.0.0.0`，控制台同时显示本机局域网 IP 对应的访问地址。

## 安全要求

- 默认监听 `127.0.0.1`。
- 所有 API 请求都必须校验启动 token。
- WebSocket 连接也必须校验 token。
- 校验请求 Origin，仅允许当前服务页面访问。
- 密码、私钥、passphrase 不明文写入配置。
- 上传下载尽量使用流式处理，不在本地长期保存临时文件。
- 删除连接时同步删除对应 vault 凭据。
- 设置页提供清空全部保存凭据的操作，并要求二次确认。

## 连接管理

连接管理 Tab 提供：

- 已保存连接列表。
- 新增连接。
- 编辑连接。
- 删除连接。
- 一键连接。
- 连接状态显示。

新增或编辑连接时支持两种认证方式：

- 密码登录。
- 私钥登录。

私钥登录支持：

- 导入私钥内容到 vault。
- 可选 passphrase。

推荐不只保存私钥文件路径，因为文件路径移动后连接会失效。第一版将私钥内容加密保存到 `vault.enc`。

连接成功后自动切换到终端 Tab。

## 终端

终端 Tab 使用 xterm.js，并通过 WebSocket 连接后端 SSH shell。

终端数据流：

```text
键盘输入 -> xterm.js -> WebSocket -> SSH stdin
SSH stdout/stderr -> WebSocket -> xterm.js
```

终端要求：

- 支持 PTY。
- 支持窗口 resize 并同步远程 PTY 尺寸。
- 支持手动断开。
- 断线后显示中文状态。
- 切换到其他 Tab 后，终端会话保持连接。
- 浏览器刷新后不承诺恢复旧 shell 会话。

终端视觉：

- 页面整体仍使用暖色羊皮纸背景。
- 终端区域使用 Rocom 暗色模式，表现为嵌入纸质面板中的暗色魔法书页。
- 终端内容使用等宽字体。
- UI 标签、按钮和状态栏使用中文圆体。
- 光标、选区和激活状态使用金色。

## 远程文件

远程文件 Tab 只操作远程服务器文件，不做本地双栏。

功能：

- 显示当前远程路径。
- 显示文件和文件夹列表。
- 进入文件夹。
- 返回上级目录。
- 上传单个文件到当前目录。
- 下载单个远程文件。
- 显示文件大小、修改时间和类型。
- 显示上传/下载进度。
- 显示权限不足、路径不存在、连接失败等错误。

实现方式：

- 文件列表通过 SFTP `readdir` 获取。
- 上传通过 HTTP API 接收浏览器文件流，再通过 SFTP 写入远程路径。
- 下载通过 SFTP 读取远程文件流，再通过 HTTP 响应返回浏览器。
- 文件传输使用独立 SFTP 会话，传输完成后关闭。

## UI 设计

整体设计必须遵循 Rocom Grimoire 设计系统：

- 暖色羊皮纸页面背景。
- 纸质浮层面板。
- 暗黑奇幻终端区域。
- 金色主操作与激活态。
- 中文圆体风格字体，回退到微软雅黑。
- 暖棕色阴影。
- 圆角控制在 8px 到 12px。
- 错误、警告、成功、信息状态使用 Rocom 功能色。

Tab 结构：

```text
连接管理 | 终端 | 远程文件 | 设置
```

连接管理页：

- 使用纸质连接卡片或紧凑列表。
- 主操作按钮使用金色。
- 危险操作使用红色软底和二次确认。

终端页：

- 顶部显示当前连接名、username、host、port 和连接状态。
- 右侧提供断开按钮。
- 主体为暗色终端面板。
- 底部显示尺寸、状态或最近错误。

远程文件页：

- 顶部为路径面包屑和上传按钮。
- 主体为文件表格。
- 文件夹和文件用图标区分。
- 下载操作放在行内按钮。

设置页：

- 显示当前监听地址。
- 显示数据目录。
- 显示 token 状态。
- 提供清空凭据功能。

## 错误处理

需要覆盖的错误：

- SSH 认证失败。
- SSH 连接超时。
- host 不可达。
- 私钥格式错误。
- passphrase 错误。
- SFTP 不可用。
- 远程路径不存在。
- 远程权限不足。
- 上传中断。
- 下载中断。
- token 缺失或错误。

错误提示使用中文，优先给出可操作信息。
